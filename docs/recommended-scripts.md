# 推荐话术

## 电话联系候选人

### 首次接通

你好，请问是「候选人姓名」吗？  
我这边是浏览器自动运营 Agent / RAG / 通用 Agent 平台方向实习岗位的面试安排人。我们看了你的简历，觉得你在浏览器自动化、Agent 平台、RAG 或工作流方向的经历和岗位比较匹配，想简单确认一下你现在是否还在看实习机会。

如果候选人表示还在看：

方便确认几个信息吗？

1. 你最早什么时候可以开始实习？
2. 每周可以实习几天？
3. 大概可以持续几个月？
4. 你简历里的浏览器自动化、Agent 平台、RAG 或工作流项目，技术面时可以展开讲吗？

收尾：

好的，我这边记录一下。如果时间合适，我们会安排一次技术面，主要聊你的项目、浏览器自动化/Agent 平台能力、RAG、工作流和工程落地。稍后我会通过短信或邮件把面试安排发给你。

### 未接电话后的短信

你好，我是浏览器自动运营 Agent / RAG / 通用 Agent 平台方向实习岗位的面试安排人。看到你的简历和岗位方向比较匹配，刚才电话联系未接。想确认你目前是否还在看实习机会，以及是否方便安排一次技术面。方便的话可以回复我，谢谢。

### 候选人不方便接电话

没关系，你方便的时候可以回拨，或者直接短信回复这几个信息：是否还在看实习、最早到岗时间、每周可实习天数、可持续多久。我们再看是否安排技术面。

## 邮件确认面试

标题：

浏览器自动运营 Agent / RAG / 通用 Agent 平台方向实习岗位面试邀请

正文：

你好，

我们看了你的简历，觉得你的浏览器自动化、RAG、Agent 平台或工作流相关经历和岗位方向比较匹配，想邀请你参加一次技术面试。

面试内容主要包括：

1. 简历项目深挖
2. 浏览器自动化、RAG 链路设计与优化
3. 通用 Agent 平台、工具调用、记忆、工作流和工程落地
4. Coze/Dify/Claw 类平台或 LLM 部署、训练相关了解情况

请回复你方便的面试时间段，我们再确认具体安排。

谢谢。

## 微信确认

你好，我是刚才电话联系你的面试安排人。  
这边同步一下岗位方向：浏览器自动运营 Agent / RAG / 通用 Agent 平台。技术面主要会聊你的项目经历、浏览器自动化、RAG、Agent 工作流和工程落地。  
请你发一下方便面试的时间段，我来协调安排。

## 面试开场

你好，今天面试大概 60 分钟。我们主要不考八股，重点围绕你的项目展开，看看你对浏览器自动化、RAG、Agent Runtime/工作流、工具调用和工程落地的理解深度。  
如果有你不确定的地方可以直接说不确定，我们更关注你怎么拆问题、怎么做取舍。

## 项目深挖转场

你选一个最熟的浏览器自动化、Agent 平台、RAG 或工作流项目来讲就行。  
先不用讲太多背景，重点讲清楚：用户输入进来后，系统怎么处理，经过哪些模块，最后怎么输出结果。

## 追问真实性

这个模块具体是你自己写的吗？  
如果我现在看代码，你会让我先看哪几个文件？  
你提到的指标是怎么测出来的？测试集有多少条？有没有 baseline？

## 追问设计能力

如果这个项目要上线给 100 个真实用户用，你最担心哪三个问题？  
如果只能保留一周内能上线的 MVP，你会砍掉哪些功能？  
如果线上回答质量突然下降，你会从哪里开始排查？

## 面试收尾

我这边主要问题问完了。你可以问一下岗位、团队或者后续流程。  
我们会综合项目深度、工程能力、实习时间和岗位匹配度做判断，后续会再同步结果。

## 通过后通知

你好，你这轮技术面反馈不错，我们想继续推进下一轮。下一轮会更偏系统设计和项目深挖，重点看浏览器自动化、通用 Agent 平台、RAG / 工作流的工程落地能力。请发一下你接下来方便的时间段。

## 暂缓或不合适通知

你好，感谢你参加这次面试。综合这次沟通和岗位要求，我们这边暂时不继续推进。主要原因是当前岗位更需要浏览器自动化、通用 Agent 平台、RAG 或工作流工程落地经验更强的候选人。祝你后续求职顺利。
