## 面试目标

验证候选人是否真的具备：

- RAG：文档解析、切分、召回、重排、上下文组装、评测与优化能力。
- 浏览器自动运营 Agent：Playwright/Selenium、账号态、抓取、发布、反爬、人工接管。
- 通用 Agent 平台：Runtime、Harness、工具注册、上下文工程、checkpoint/resume、权限审计。
- 工作流平台：能否设计可控的状态流、节点、重试、兜底、Coze/Dify/LangGraph 取舍。
- Agent：工具调用、记忆、权限、安全边界、可观测和工程落地能力。
- 加分项：了解 LLM 部署、训练、蒸馏、RL 后训练。

## 60 分钟流程

| 时间 | 模块 | 目标 |
| --- | --- | --- |
| 0-5 分钟 | 开场确认 | 到岗时间、实习周期、最熟项目 |
| 5-20 分钟 | 项目深挖 | 验证个人职责、项目深度、指标真实性 |
| 20-35 分钟 | 方向专项 | 浏览器自动化 / Agent Runtime / RAG / 工作流按项目选择 |
| 35-50 分钟 | 工程落地 | 看权限、审计、checkpoint、失败恢复、测试和部署 |
| 50-55 分钟 | 加分项 | 部署、训练、蒸馏、RL 后训练 |
| 55-60 分钟 | 候选人提问 | 观察表达、关注点和沟通方式 |

## 九、现场设计题

题目：设计一个“浏览器自动运营 Agent + 通用 Agent 工作台”。

要求候选人覆盖：

- 浏览器登录、抓取、表单操作、内容发布和人工接管。
- Playwright/Selenium 执行器、截图、DOM 解析、等待策略和异常恢复。
- 工具注册、权限控制、审计日志和风险操作审批。
- Agent Runtime、上下文工程、短期/长期记忆和 checkpoint/resume。
- RAG 素材库、历史案例库、运营知识库的检索、重排和引用。
- Coze/Dify 快速搭建和自研 LangGraph/Runtime 的边界。
- 评测指标、线上监控、成本和延迟控制。

追问：

1. 如果浏览器任务执行到一半页面结构变了怎么办？
2. 如果自动发布前发现内容命中敏感词或低置信度怎么办？
3. 如果同一账号连续失败 3 次，系统如何降级和保护账号？
4. 如果老板要求 1 周上线 MVP，你会砍掉哪些功能？
5. 如果线上成本或封号风险超预算，你先优化哪三个点？
