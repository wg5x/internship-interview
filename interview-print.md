# AI Agent / RAG 实习生面试题单

候选人：__________  面试官：__________  时间：__________

## 面试目标

验证候选人是否真的具备：

- RAG：文档解析、切分、召回、重排、上下文组装、评测与优化能力。
- 工作流：能否设计可控的状态流、节点、重试、兜底、checkpoint。
- Agent：工具调用、记忆、权限、安全边界、可观测和工程落地能力。
- 加分项：了解 LLM 部署、训练、蒸馏、RL 后训练。

## 60 分钟流程

| 时间 | 模块 | 目标 |
| --- | --- | --- |
| 0-5 分钟 | 开场确认 | 到岗时间、实习周期、最熟项目 |
| 5-20 分钟 | 项目深挖 | 验证个人职责、项目深度、指标真实性 |
| 20-35 分钟 | RAG 专项 | 看是否懂完整链路和优化方法 |
| 35-50 分钟 | Agent / 工作流 | 看是否能设计可控、可落地的 Agent |
| 50-55 分钟 | 加分项 | 部署、训练、蒸馏、RL 后训练 |
| 55-60 分钟 | 候选人提问 | 观察表达、关注点和沟通方式 |

## 三、RAG 专项

必问题：

1. 一个企业知识库 RAG，从文件上传到最终回答，完整链路有哪些步骤？
2. PDF、Word、网页、图片混合文档怎么解析？什么时候需要 OCR 或 VLM？
3. chunk size、overlap、父子分块、语义分块分别解决什么问题？你怎么选？
4. BM25、向量检索、混合检索、RRF、Cross-Encoder Rerank 的作用和代价是什么？
5. 如何判断一次回答不好，是召回问题、排序问题、上下文组装问题，还是生成问题？

进阶追问：

1. Query Rewrite、HyDE、Multi-Query 分别适合什么场景？有什么风险？
2. 如何设计 RAG 评测集？Hit@K、MRR、context precision、faithfulness 各看什么？
3. RAG 回答出现幻觉，你会怎么定位和修复？
4. 多轮对话里，历史消息和检索上下文怎么拼接，避免 token 爆炸？

## 四、Agent / 工作流专项

必问题：

1. ReAct、Plan-and-Execute、Reflection、Multi-Agent 分别适合什么任务？
2. 用 LangGraph 设计一个“问答 + 查数据库 + 建工单”的状态图，节点和边怎么定义？
3. 工具调用的 schema 怎么设计？如何处理参数缺失、参数错误、工具超时？
4. 工具调用失败后，Agent 应该重试、换工具、问用户还是兜底？怎么判断？
5. Agent 的短期记忆、长期记忆、用户画像和任务状态有什么区别？

进阶追问：

1. 什么情况下需要多 Agent？多 Agent 相比单 Agent 的问题是什么？
2. checkpoint / resume 在长任务里解决什么问题？哪些状态必须持久化？
3. 如何防止 Agent 调危险工具、越权访问、重复调用或泄露敏感信息？
4. 如何做 Agent 可观测性？你会记录哪些 trace 字段？

## 七、现场设计题

题目：设计一个企业内部“知识库问答 + 工单处理 Agent”。

要求候选人覆盖：

- 文档解析和知识库入库。
- 混合检索和重排。
- 用户意图识别和工作流路由。
- 工具调用和权限控制。
- 多轮会话记忆。
- 失败重试和人工兜底。
- 评测指标和线上监控。
- 部署方案和成本优化。

追问：

1. 如果召回结果为空怎么办？
2. 如果工具返回数据和文档知识冲突怎么办？
3. 如果用户要求删除知识库中的隐私数据怎么办？
4. 如果老板要求 1 周上线 MVP，你会砍掉哪些功能？
5. 如果线上成本超预算，你先优化哪三个点？
